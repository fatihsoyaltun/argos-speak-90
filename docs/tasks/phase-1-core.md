# TASK: Phase 1 — Core Skeleton

## Context
Read AGENTS.md first before doing anything.

## Goal
Build a working mobile-first core skeleton for Argos Speak 90.

This phase is not about full learning logic yet.
This phase is about creating a clean and stable base.

## Product Direction
This is a personal English speaking trainer.
It should feel focused, premium, calm, and easy to use every day.

## Tech
- Next.js
- TypeScript
- App Router
- Tailwind CSS

## Hard Rules
- Mobile-first
- No backend
- No auth
- No database
- No feature creep
- No overdesigned landing page
- No childish UI
- Rounded, premium UI only

## Pages to Create
Create these pages:
- /
- /today
- /listen
- /words
- /speak
- /review
- /stats
- /settings

## Layout Requirements
- Sticky top area or stable app shell
- Clean bottom navigation
- Strong mobile usability
- Rounded cards
- Clear section spacing
- Good button styling
- Good typography hierarchy

## Home Page Purpose
The root page should be simple.
It can briefly explain what the app is and route the user into Today.

## Today Page Purpose
This should feel like the command center.
It should clearly show:
- current day
- today’s flow
- what to do next

## Listen Page Purpose
This page should exist as a clean placeholder for future listening rebuild.

## Words Page Purpose
This page should exist as a clean placeholder for future words upgrade.

## Speak Page Purpose
This page should exist as a clean placeholder for future speaking upgrade.

## Review Page Purpose
This page should exist as a clean placeholder for future review upgrade.

## Stats Page Purpose
This page should exist as a clean placeholder for future progress UI.

## Settings Page Purpose
This page should exist as a clean placeholder for utilities and app settings.

## Design Direction
Use a premium mobile-first visual system:
- rounded components
- calm but strong colors
- polished buttons
- readable text
- no clutter

## Deliverables
Build:
- app shell
- page routes
- bottom navigation
- placeholder screens with clear purpose
- reusable shared components if helpful

## Do Not Add Yet
- backend
- auth
- database
- timers
- speech engine
- content system
- streak logic
- daily storage logic

## After Changes
1. Run lint
2. List changed files
3. Summarize what was built
4. Explain what should be checked next
