# Curriculum Repair Summary

## Purpose

This document summarizes the Phase Quality-2 curriculum repair pass completed after `docs/CURRICULUM_AUDIT.md`.

The goal of the repair was to make Argos Speak 90 more credible as a speaking-first 90-day path toward functional early B1 in familiar everyday/work contexts, without redesigning the app, changing UI structure, changing Supabase/Auth/Admin logic, adding ForenScope/device-specific content, or expanding beyond Day 90.

## Audit Risks Addressed

The repair pass addressed these risks from the curriculum audit:

- Review was too exact-answer based.
- Later speaking prompts were too template-driven.
- Vocabulary boosters repeated too mechanically.
- Repair and clarification language was underdeveloped.
- Milestone assessment was too weak.
- Later listening tasks stayed too easy.

## What Changed

- Every day now has one production-style review item. These items ask the learner to write, personalize, clarify, summarize, add a reason, correct a misunderstanding, or suggest a next step.
- Days 15-90 speaking tasks now have more variety, including mini role-play, short message replies, issue explanation, clarification, soft opinion, comparison, summary, and follow-up question tasks.
- Vocabulary now prioritizes theme-aware chunks before general stage boosters, while still keeping exactly 8 vocabulary items per day.
- Repair and clarification language was added naturally across Words, Speak, and Review, including phrases like `I mean`, `What I meant was`, `Could you clarify that?`, `Let me check one detail`, `In other words`, and `To be more specific`.
- Milestone days 14, 42, 70, and 90 were strengthened to test output length, target chunks, reasons/examples, weak-point awareness, and second-try improvement.
- Days 43-90 listening outputs were scaled up so learners identify the main point, apply useful phrases, summarize briefly, give a reaction, or connect the listening to their own life/work.

## Remaining Limitations

- There is no live teacher feedback.
- There is no automatic spoken grammar or pronunciation scoring.
- Review checking is still simple and model-answer based, so good open-ended answers may still need human judgment.
- ForenScope/device-specific English has not been added yet.
- Real pilot data is still needed to confirm difficulty, usefulness, completion rate, and speaking output quality.
- Assessment is now clearer, but the pilot still depends on honest self-check and admin review of synced written answers.

## Final Verdict

The curriculum is now stronger and more credible as early B1-oriented speaking practice and as a speaking-first path toward functional early B1 in familiar everyday/work contexts.

It should still be validated through real pilot use and external AI review, and it should not be presented as a guaranteed full B1 outcome.
