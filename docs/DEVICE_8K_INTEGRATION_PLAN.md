# Device 8K Micro Integration Plan

## Purpose

This document plans how verified ForenScope 8K Android Latent Fingerprint Detection Tablet English can be integrated into Argos Speak 90 later, without turning the 90-day speaking program into a device brochure.

This is a planning document only. It does not change app content, curriculum files, UI, Supabase, Auth, Admin, or cloud sync logic.

## Sources

- `docs/DEVICE_ENGLISH_SOURCEBOOK.md`
- `docs/ACTUAL_CURRICULUM_VERIFICATION.md`
- `docs/CURRICULUM_REPAIR_SUMMARY.md`

## 1. Integration Principle

Argos Speak 90 should remain a general, speaking-first English program. The 8K device content should appear only as small work/technical micro-contexts on selected mid-to-late days where the existing curriculum already practices clarification, feedback, process explanation, polite instructions, reporting, decisions, and summaries.

Core rules:

- Keep the main 90-day curriculum general speaking-first English.
- Add 8K device English only as micro-context in selected work/technical days.
- Do not overload early A1 days.
- Do not add unsupported device facts.
- Keep all device wording source-backed and cautious.
- Limit each selected day to a maximum of 2-3 new device terms.
- Use simple English first; keep Turkish explanations clear and practical.
- Use cautious wording such as `can support`, `may help`, `appears clearer`, `observed`, and `source states`.
- Avoid guarantee-style wording such as `always`, `guarantees`, `legally proven`, or `certified` unless a future verified source directly supports it.

## 2. Why Micro-Context Fits the Current Curriculum

The verified curriculum already supports early B1-oriented speaking practice in familiar everyday/work contexts:

- Days 43-90 listening prompts ask learners to identify main points, summarize, react, apply phrases, and connect tasks to real work/life.
- Days 15-90 speaking tasks include role-play, short message replies, issue explanation, clarification, soft opinions, comparison, summaries, and follow-up questions.
- Every day has one production-style review item.
- Milestone days 14, 42, 70, and 90 test output length, target chunks, reasons/examples, weak-point awareness, and second-try improvement.

The 8K material should therefore support those existing speaking functions rather than replace them.

## 4. Technical Term Limits

Each selected day should introduce only 2-3 device terms. Existing general English should remain the main learning target.

Recommended term discipline:

- One concrete noun: `filter`, `darkroom`, `report`, `archive`, `camera`.
- One workflow phrase: `contactless workflow`, `AFIS workflow`, `required format`.
- One cautious reporting phrase: `appears clearer`, `was observed`, `may help`.

Avoid adding full feature lists inside daily content. Full lists can stay in the sourcebook.

## 5. Protecting the B1 Progression

The 8K micro-context should strengthen the learner's speaking functions:

- Clarification: "Could you check the selected filter?"
- Polite instruction: "Please use the darkroom when working with UVC light."
- Process explanation: "First select the light, then check the filter."
- Reporting: "The fingerprint was observed on thermal paper."
- Decision-making: "I think we should export it as WSQ because..."
- Customer demo language: "The system can support contactless fingerprint imaging."

The device content should not become a memorization list. Learners should use device words to ask, explain, decide, summarize, and report.

## 6. Risk Section

### Risk: Overloading Learners

Adding too many device terms can make mid-level days feel like technical vocabulary drills.

Mitigation:

- Use only eight selected days.
- Limit each day to 2-3 device terms.
- Keep every task short and output-based.
- Use simple Turkish explanations for each device word.

### Risk: Unsupported Technical Claims

Device phrases can accidentally become stronger than the PDF supports.

Mitigation:

- Use only sourcebook terms and page references.
- Keep cautious wording: `can support`, `may help`, `appears clearer`, `observed`, `source states`.
- Avoid guarantee language.
- Do not add repair, maintenance, legal, certification, or performance claims.

### Risk: Making Argos Too ForenScope-Specific

Too much 8K content could weaken the general English path.

Mitigation:

- Keep 8K as micro-context only.
- Do not add device content to early A1 days.
- Keep normal speaking targets intact.
- Do not add a separate device-heavy track inside the 90-day base program yet.

### Risk: Technical Terms Without Speaking Output

Learners may memorize terms but not use them in conversation.

Mitigation:

- Every selected day should include a speaking action: clarify, instruct, explain, report, decide, or summarize.
- Review tasks should require a short personalized or scenario-based sentence.
- Do not add passive reading-only device facts.
